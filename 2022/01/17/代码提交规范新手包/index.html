prettier代码美化

  • 安装

lint-stagehttps://github.com/okonet/lint-staged#examples

yarn add prettier --dev

// 为了不与eslint冲突
yarn add eslint-config-prettier

// 初始化lint-stage
npx mrm@2 lint-staged

// 这里会自动安装husky
  • 创建配置文件 .prettierrc .prettierIgnore
  • 修改 eslint 配置
<!-- package.json -->
  "eslintConfig": {
  "rules": {
    "@typescript-eslint/no-unused-vars": "off"
  },
  "extends": [
    "react-app",
    "react-app/jest",
    "prettier" ++
  ]
},

"lint-staged": {
  "*.{js,css,md,ts,tsx}": "prettier --write"
}

commit 校验工具

  • 安装
// 安装cli和配置
yarn add @commitlint/{config-conventional,cli} --dev

// 创建配置commitlint文件
echo "module.exports = {extends:['@commitlint/config-conventional']}" > commitlint.config.js
  • 配置

// 添加husky钩子
npx husky add .husky/commit-msg 'npx --no-install commitlint --edit "$1"'

commit 交互式提交工具

https://commitlint.js.org/#/?id=getting-started

  • 安装

    npm i -d commitizen

  • 预设配置

    // 预设
npm i -d cz-conventional-changelog // or cz-conventional

  • 修改package.json

    // 添加config
"config": {
  "commitizen": {
    "path": "cz-conventional-changelog // cz-conventional"
  }
}

  • 交互式提交

    // 添加到script
"commit":"cz"

生成 CHANGELOG

  • 安装

    npm i -D standard-version

  • 添加 script

    {
  "scripts": {
    "release": "standard-version"
  }
}

  • 运行后会自动生成CHANGELOG.md，当提交类型是 feat 和 fix 的时候版本号会累加
