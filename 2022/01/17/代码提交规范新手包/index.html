代码提交规范新手包

本文最后更新于：2022年4月4日 晚上

commit 提交工具

  • 安装

    npm i -d commitizen

  • 预设配置

    // 预设
npm i -d cz-conventional-changelog

  • 修改package.json

    // 添加config
"config": {
  "commitizen": {
    "path": "cz-conventional-changelog"
  }
}

  • 交互式提交

    // 添加到script
"commit":"cz"

//或者直接
cz

commit 校验工具

  • 安装

    // husky - git的钩子工具
// commitlint - 校验工具

npm i -d husky commitlint

  • 配置

    npm i -d @commitlint/config-conventional

//  .commitlintrc.js
module.exports = {
  extends: 
  // angluar 规范校验
  ['@commitlint/config-conventional']
};
    // 初始化husky
npx husky install

// 新建文件 .husky/commit-msg

#!/bin/sh

npx --no-install commitlint --edit "$1"

  • 在commit之前执行操作

    npx husky add .husky/pre-commit 'npm run lint'
// 会在commit之前执行eslint

  • 之后的所有commit信息都将校验并抛出校验错误结果

生成CHANGELOG

  • 安装

    npm i -D standard-version

  • 添加script

    {
  "scripts": {
    "release": "standard-version"
  }
}

  • 运行后会自动生成CHANGELOG.md，当提交类型是feat和fix的时候版本号会累加
